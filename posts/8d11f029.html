HTB-Manager

发表于 2025-03-24|更新于2025-03-26|HackTheBox
|字数总计:3005|阅读时长:17分钟|阅读量:

靶机ip：10.10.11.236

知识点

  • SMB RID爆破
  • 用户名密码爆破
  • xp_dirtree 枚举目录
  • ESC7提权

信息收集

HTTP服务

是个静态网站，目录扫描也没有结果，暂时先搁置了

SMB服务

发现没什么有价值的共享

使用netexec进行RID枚举

发现很多用户名，并且保存到smb_rid_brute文件中了，将用户名提取出来

将结果保存到user.txt中

根据用户名制作一个简单的字典

  • 用户名全大写
  • 用户名全小写
  • 用户名与密码相同

使用netexec枚举

得到了一组用户名密码

继续使用netexec进行枚举

smb没什么有价值的东西，继续枚举其他协议

发现可以利用mssql和wmi，尝试后发现使用impacket-wmiexec登陆失败，原因是权限不足，所以我们利用mssql登录

注：需要加-windows-auth参数才可以登录

MSSQL攻击

  • xp_cmdshell 执行命令失败
  • xp_dirtree 捕获hash也无法破解成明文

事已至此只能使用xp_dirtree查找一下机器的文件了

最终在C:\inetpub\wwwroot下找到了备份文件

从网站上下载下来

发现有一个.old-conf.xml文件

找到了另一组用户名密码

使用netexec继续枚举

使用evil-winrm登录

成功登录，并在Desktop找到user.txt

ESC7提权

发现这两个用户都是域用户，并且都配置了ADCS服务

使用certipy-ad寻找利用点

发现可以使用ESC7提权

参考链接：https://www.thehacker.recipes/ad/movement/adcs/access-controls#esc7-abusing-subca

第一步：通过 ManageCa 权限，赋予自己 ManageCertificates 权限 (使用 -add-officer 参数)

第二步： 使用 SubCA 证书模板，注册一个 SAN 为 Administrator 的证书

记下我们的 Request ID 并 (输入 y) 保留私钥

第三步：通过 ManageCertificates 权限发布刚刚申请失败的证书

第四步：通过 ID 和私钥文件，检索我们发布的证书

第五步：使用证书进行认证，拿到 Administrator 账户的 NT hash

先同步一下时间

继续攻击

使用netexec验证一下hash

使用evil-winrm登录

登陆成功，在Desktop找到root.txt
